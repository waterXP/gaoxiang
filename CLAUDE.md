## Architecture

### Data Pipeline

Raw OCR output (`ocr_full_output.json`) в†’ `build_site.py` в†’ `site/data.js` в†’ `split_data.js` в†’ per-chapter JS files in `app/src/data/chapters/`.

The React app imports chapter data statically (no runtime fetch). When re-running OCR or adding new chapters, run both pipeline scripts then rebuild.

### App Structure (`app/src/`)

- **`data/chapters/ch_XX.js`** вЂ” Per-chapter study content (ch_00 through ch_36). Each exports an array of page objects. These are the primary files to edit when updating content.
- **`data/meta.js`** вЂ” Chapter metadata (title, num, startPage). Manually editable to fix chapter names.
- **`data/index.js`** вЂ” Auto-generated barrel of chapter imports. Regenerated by `split_data.js`; do not edit manually.
- **`data/types.js`** вЂ” Content type definitions with colors/icons: `й‡Ќз‚№`, `йљѕз‚№`, `иЂѓз‚№`, `е®љд№‰`, `е…¬ејЏ`, `жЎ€дѕ‹`, `ж™®йЂљ`.
- **`hooks/useStorage.js`** вЂ” All user state (chapter progress, daily streak) persisted to `localStorage` under key `'sp'`.
- **`utils/renderContent.jsx`** вЂ” Renders structured `content` array format (alternative to raw `html` strings).

### Page Data Format

Each entry in a `ch_XX.js` file:

```js
{
  page: 42,
  type: "й‡Ќз‚№",          // from types.js keys
  color: "#ff6b6b",     // optional override
  html: `<h2>...</h2><p>...</p>`,   // raw HTML string, OR
  content: [            // structured alternative to html
    { tag: "h2", text: "..." },
    { tag: "p", parts: [{ text: "red", color: "#f00" }] },
    { tag: "ul", items: ["item1", { text: "item2", bold: true }] },
    { tag: "table", headers: ["A", "B"], rows: [["c1", "c2"]] },
  ]
}
```

A page uses either `html` or `content`, not both.

### Key Components

- **`App.jsx`** вЂ” Root state: current chapter index, memo mode, search, streak. Builds Fuse.js search index from all pages at load time.
- **`ChapterView.jsx`** вЂ” Renders a chapter's pages as `PageBlock` components. Memo mode hides `<p>` and `<li>` text until clicked.
- **`Sidebar.jsx`** вЂ” Chapter list with progress tracking and daily check-in button.

### Deployment

Push to `main` triggers GitHub Actions (`.github/workflows/deploy.yml`) which runs `npm ci && npm run build` in `app/` and deploys `app/dist/` to GitHub Pages. The Vite base path is `/gaoxiang/`.
